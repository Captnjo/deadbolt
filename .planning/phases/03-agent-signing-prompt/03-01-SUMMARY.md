## Decisions Made

- `stream_intents` body comments out `sink.add(_event)` — `SseEncode` for `IntentEvent` is generated by FRB codegen; the function signature is correct and the receiver-claiming logic compiles cleanly; the actual forwarding loop is enabled after `flutter_rust_bridge_codegen generate` runs
- `tokio` with `features = ["sync"]` added to bridge Cargo.toml — FRB provides the async runtime but the `mpsc::UnboundedReceiver` type requires the sync feature explicitly
- `crate::frb_generated::StreamSink` used instead of `flutter_rust_bridge::StreamSink` — the latter was removed from FRB 2.x root module; `frb_generated_boilerplate!` macro re-exports it via `crate::frb_generated`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed non-exhaustive match in guardrails.rs**
- **Found during:** Task 1 (AddSignMessage to IntentType)
- **Issue:** `intent_mint()` function in `guardrails.rs` had an exhaustive match on `IntentType` variants. Adding `SignMessage` caused a compile error: `non-exhaustive patterns: &IntentType::SignMessage { .. } not covered`
- **Fix:** Added `super::intent::IntentType::SignMessage { .. } => None` arm — `SignMessage` has no token mint, so returns `None` like `SendSol`
- **Files modified:** `rust/deadbolt_core/src/agent/guardrails.rs`
- **Verification:** `cargo test -p deadbolt_core agent::intent` passes
- **Committed in:** `53a886e` (Task 1 commit)

**2. [Rule 3 - Blocking] Added tokio dependency to bridge Cargo.toml**
- **Found during:** Task 1 (building bridge with INTENT_RX static)
- **Issue:** `use tokio::sync::mpsc` failed — tokio not in `deadbolt_bridge/Cargo.toml`
- **Fix:** Added `tokio = { version = "1", features = ["sync"] }` — minimal feature set needed for the type
- **Files modified:** `rust/deadbolt_bridge/Cargo.toml`
- **Verification:** `cargo build -p rust_lib_deadbolt` exits 0
- **Committed in:** `53a886e` (Task 1 commit)

**3. [Rule 3 - Blocking] Fixed StreamSink import path and stream_intents body for compilation**
- **Found during:** Task 1 (building bridge with stream_intents)
- **Issue:** `flutter_rust_bridge::StreamSink` no longer exported at crate root in FRB 2.x; `IntentEvent` doesn't implement `SseEncode` (generated by codegen) so `sink.add()` won't compile without codegen
- **Fix:** Changed import to `crate::frb_generated::StreamSink`; stub out `sink.add()` call with a comment until FRB codegen runs and implements `SseEncode` for `IntentEvent`
- **Files modified:** `rust/deadbolt_bridge/src/api/agent.rs`
- **Verification:** `cargo build -p rust_lib_deadbolt` exits 0 with warnings only
- **Committed in:** `53a886e` (Task 1 commit)
