### Root Cause Analysis for Gaps

**Gap cluster 1 (AGNT-06, AGNT-08 partial): Missing unsigned transaction infrastructure**
Both simulation and sign_message approval require Rust-side transaction building functions that were never created:
- `buildUnsignedSendSol` — needed for simulation
- `buildUnsignedSendToken` — needed for simulation
- `sign_message` — needed for signing message bytes with wallet's Ed25519 key

These are separate Rust bridge functions that must be added to `deadbolt_bridge` and exposed through FRB codegen.

**Gap cluster 2 (signMessage stub wiped by codegen)**
The SUMMARY for Plan 02 documented adding a `signMessage` stub to `agent.dart`. However, `agent.dart` is FRB-generated — codegen subsequently regenerated it and erased the stub. The correct fix is to add `sign_message` as a Rust function in `deadbolt_bridge` so that FRB codegen generates the Dart binding.

### Gaps Summary

The phase delivered a complete and functional agent signing prompt system for `send_sol` and `send_token` intents — the two primary use cases. The Rust infrastructure (SignMessage enum variant, broadcast channel, all FRB bridge functions), the Flutter state layer (IntentNotifier, 3 providers, signed tx pipeline for SOL/token), and the full UI layer (bottom sheet with 5 intent preview types, queue section, badge, auto-show) are all verified and properly wired.

Two gaps block full goal achievement:

**Gap 1 — sign_message signing unimplemented (blocker for AGNT-08 + AGNT-03 completeness):** The `sign_message` bridge function was documented as added in Plan 02 but the target file (`agent.dart`) is FRB-generated and codegen subsequently wiped it. A Rust-side `sign_message` function must be added to `deadbolt_bridge` and codegen re-run so the Dart binding is stable.

**Gap 2 — simulation always idle (AGNT-06 unmet):** The `_runSimulation` function unconditionally sets `SimulationPhase.idle` because the unsigned transaction builder does not exist in the FRB bridge. The UI gracefully hides the simulation row when idle, so no crashes occur — but AGNT-06 ("simulation runs in background; prompt updates with results") is not fulfilled.

Both gaps share the root cause: Rust bridge functions were not built for transaction building/message signing. These can be addressed as a focused plan adding the missing Rust functions.
