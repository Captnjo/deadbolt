## Spacing Scale

Declared values (must be multiples of 4). Source: observed in agent_api_screen.dart, send_screen.dart, app_shell.dart.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, status dot spacing |
| sm | 8px | Between stacked labels, between icon and text in rows |
| md | 16px | Within-section spacing, dialog content gaps |
| lg | 24px | Screen-edge padding, section-level vertical spacing |
| xl | 32px | Between major sections (e.g. after a Divider) |
| 2xl | 48px | Empty state top/bottom breathing room |
| 3xl | 64px | Not used in Phase 3 screens |

Exceptions:
- Bottom sheet drag handle: 4px height, 32px wide, centered — does not fall on 8-point scale; use 4px height as design exception for drag affordance
- Touch targets for Approve/Reject buttons: minimum 48px height (accessibility requirement, macOS pointer target minimum)
- NavigationRail badge label: uses Material 3 Badge widget default — font size is system-controlled and not part of the project type scale

### 1. SigningPromptSheet (`lib/features/agent/signing_prompt_sheet.dart`)

Bottom sheet. Implemented as `showModalBottomSheet` + `DraggableScrollableSheet`.

| Property | Value |
|----------|-------|
| Container color | `#1E1E1E` (card) |
| Border radius (top) | 16px top-left + top-right |
| Border | 1px `#2A2A2A` top edge only |
| Initial height | 60% of screen (`initialChildSize: 0.6`) |
| Min height | 40% (`minChildSize: 0.4`) |
| Max height | 90% (`maxChildSize: 0.9`) |
| Dismiss behavior | `isDismissible: true` — swipe down or tap backdrop = dismiss, intent stays pending |
| `isScrollControlled` | true (required for DraggableScrollableSheet) |
| Background | `Colors.transparent` (sheet draws its own container) |

Drag handle:
- Centered horizontally at top of sheet
- 4px tall × 32px wide
- Color: `#2A2A2A` (border)
- 12px top padding above handle, 16px padding below handle before content

"N more pending" indicator:
- Displayed below the drag handle when `pendingCount > 1`
- Text: "N more pending" — 12px, textSecondary, centered
- No visual chip or badge — plain text, italic acceptable

### 2. IntentPreviewContent (inline widget within SigningPromptSheet)

Preview rows follow the same pattern as send confirmation screen: label row (14px textSecondary) above value row (14px textPrimary), separated by 4px.

Shared preview rows (all intent types):
- **Agent**: "Claude agent" or token prefix label — 12px label, monospace, textSecondary
- **Request type**: intent type as heading (24px bold)
- **Network fee**: "~0.000005 SOL" — 14px body

Type-specific rows:

**SendSol / SendToken:**
- Amount: large display — 24px bold, accent color for SOL/token value; USD equivalent on same row at 14px textSecondary
- Recipient: 14px, truncated middle (first 6 + "..." + last 4 chars), monospace

**Swap:**
- Input: amount + token symbol — 14px bold
- Output: "~X TOKEN" — 14px, textSecondary, with "Fetching..." placeholder if quote not yet loaded
- Exchange rate: 12px textSecondary
- Slippage: 12px textSecondary
- Quote loading state: 12px textDisabled "Fetching quote..."
- Quote unavailable: 12px warning color "Quote unavailable"

**SignMessage:**
- Info banner: full-width container, `#E2A93B` at 10% opacity background, 1px `#E2A93B` border, 8px radius — copy: "Message signature only. No transaction, no fees."
- Message body: 12px monospace, textSecondary, scrollable within sheet, max 6 lines before scroll. Label: "Message (UTF-8)" or "Message (hex)" — 12px textSecondary
- No fee row (no network fee for message signing)

### 3. SimulationStatusRow (inline in SigningPromptSheet)

Positioned directly below preview rows, above action buttons.

| State | Visual |
|-------|--------|
| Running | `CircularProgressIndicator` (stroke 2, accent color, size 14px) + "Simulation: Running..." (12px textSecondary) |
| Success | `Icons.check_circle_outline` (14px success green) + "Simulation: Passed" (12px success green) |
| Failed (pre-approve) | Red warning banner: `#E74C3C` at 10% opacity background, 1px `#E74C3C` border, 8px radius. Header: "Simulation failed" (14px error, bold). Body: error message (12px textSecondary). Footer: "Approving a failed simulation may result in a lost tx fee" (12px warning color) |
| Idle | Hidden — no element rendered |

### 4. SigningProgressView (replaces preview content post-approve)

After approve is tapped, the preview content is replaced by a lifecycle progress view. No sheet dismiss — sheet stays open during lifecycle.

| Lifecycle step | Visual |
|----------------|--------|
| Signing | `CircularProgressIndicator` (accent, size 24px) + "Signing..." (16px bold) |
| Submitting | `CircularProgressIndicator` (accent, size 24px) + "Submitting..." (16px bold) |
| Confirmed | `Icons.check_circle` (48px success green) + "Confirmed" (20px bold) + truncated signature (12px monospace textSecondary) — auto-dismisses after 3 seconds |
| Failed | `Icons.error_outline` (48px error red) + error message (14px textSecondary) + [Retry] (outlined, primary) + [Dismiss] (text button) |

Hardware wallet variant (Signing step only):
- Replaces spinner with `Icons.usb` (24px, accent) + "Connect your ESP32 to sign" (14px body) + device detection indicator

### 5. IntentQueueSection (top of agent_api_screen.dart)

Inserted as first section in the ListView, before existing Server section. Conditionally rendered — hidden when `pendingIntents.isEmpty`.

Section header: "PENDING REQUESTS" — 14px, weight 700, textSecondary (matches existing section header pattern).

Queue row per pending intent:
- `ListTile` with `contentPadding: EdgeInsets.zero`
- Title: intent summary one-liner — 14px body — e.g. "Send 1.5 SOL", "Swap 100 USDC → SOL", "Sign message"
- Subtitle: "2 min ago" — 12px textSecondary

After queue section: `const SizedBox(height: 24)` + `const Divider()` before Server section.

## Interaction Contracts

### Signing Prompt Auto-Show

Trigger: new intent arrives via `streamIntents()` stream while app is unlocked and not displaying a modal.

Guard conditions (do NOT show if any true):
1. App is locked (`authProvider.isLocked == true`) — rely on badge only
2. A modal is currently showing (`ModalRoute.of(context)?.isCurrent == false`)

Result when shown: `showModalBottomSheet` called via root navigator key, sheet shows first pending intent.

### Dismiss Behavior

- Swipe down: dismiss sheet, intent stays `pending`, badge count unchanged
- Tap backdrop: dismiss sheet, intent stays `pending`, badge count unchanged
- Reject button: calls `reject_intent(id)`, intent removed from queue immediately, sheet closes, badge count decrements
- Approve/Sign button: transitions sheet to SigningProgressView — sheet stays open until Confirmed (auto-dismiss 3s) or Failed (user action needed)

### Queue Progression

After approving/rejecting the displayed intent:
- If queue has more pending intents: next intent slides in automatically (same sheet instance, replace content)
- If queue is empty: sheet closes

### Reject Confirmation

Rejection requires NO confirmation dialog. Tap Reject → instant rejection → sheet advances to next intent or closes. Source: CONTEXT.md decision "Rejection: instant, no reason field."

### Simulation Warning Approval

Approve button remains enabled in all simulation states. User may approve before simulation completes or after simulation failure. No blocking gate.

### Stake Intent Handling

Stake intents arrive in queue with preview showing:
- Intent type header: "Stake" (24px bold)
- Full-width info banner (warning color): "Staking is not supported in this version."
- Approve button: disabled (grayed out, `BrandColors.textDisabled` background)
- Reject button: visible and enabled
- Auto-reject via `reject_intent(id)` called immediately on arrival with error: "Staking not supported in this version"

## State Matrix

All states the signing prompt must handle without ambiguity:

| State | What User Sees |
|-------|----------------|
| No pending intents | Section hidden on Agent API screen. No badge. No sheet. |
| 1 pending intent, app active | Sheet auto-shows. Badge shows "1". |
| 3 pending intents, app active | Sheet auto-shows with first intent. "2 more pending" below drag handle. Badge shows "3". |
| App locked, intents arrive | Badge count increments. No sheet auto-show. |
| Simulation running | Spinner + "Simulation: Running..." in sheet. Approve enabled. |
| Simulation passed | Green check + "Simulation: Passed". Approve enabled. |
| Simulation failed | Red warning banner. Approve still enabled with warning. |
| Swap intent — quote loading | "Fetching quote..." in output field. Approve enabled. |
| Swap intent — quote failed | "Quote unavailable" in output field. Approve enabled. |
| Approve tapped | Preview replaced by SigningProgressView. "Signing..." with spinner. |
| ESP32 wallet active | Hardware wallet variant: USB icon + "Connect your ESP32 to sign". |
| Transaction confirmed | Green check "Confirmed" + truncated signature. Auto-dismiss 3s. Next intent or close. |
| Transaction failed | Error icon + error message + [Retry] [Dismiss]. Intent marked failed. |
| Retry tapped | Returns to "Signing..." state. |
| Reject tapped | Sheet closes or advances to next intent. Badge decrements. |
| Stake intent arrives | Queue row shows. Sheet shows with Approve disabled + unsupported banner. Auto-rejected immediately. |
