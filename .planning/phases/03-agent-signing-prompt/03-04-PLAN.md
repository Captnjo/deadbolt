  <how-to-verify>
**Prerequisites:**
1. Run FRB codegen if not already done: `flutter_rust_bridge_codegen generate`
2. Start the app: `flutter run -d macos`
3. Unlock the app with password
4. Navigate to Agent API screen
5. Create an API key if none exists
6. Toggle the agent server ON

**Test 1: Intent submission (AGNT-03)**
In a terminal, POST a send_sol intent:
```bash
curl -X POST http://localhost:9876/intent \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"type": "send_sol", "to": "11111111111111111111111111111111", "lamports": 1500000000}'
```
Expected: 201 response with `{ "id": "...", "status": "pending" }`

**Test 2: Signing prompt auto-show (AGNT-04)**
After posting the intent, the bottom sheet should auto-appear from the bottom covering ~60% of screen.
Expected: Sheet shows with:
  - Drag handle at top
  - Agent label (token prefix)
  - "Send SOL" heading (24px bold)
  - Amount: "1.5 SOL"
  - Recipient: truncated address
  - Simulation status (Running... then Passed or Failed)
  - Approve button (orange) + Reject button (red outline)

**Test 3: Dismiss behavior (AGNT-04)**
Swipe the sheet down or tap outside.
Expected: Sheet dismisses, intent stays pending, badge shows "1"

**Test 4: Queue + badge (AGNT-05)**
POST 2 more intents while the sheet is dismissed.
Expected:
  - Badge on Agent API icon shows "3"
  - Agent API screen shows "PENDING REQUESTS" section with 3 rows
  - Each row shows type summary + time ago + Review button

**Test 5: Review from queue (AGNT-05)**
Tap "Review" on any queue row.
Expected: Bottom sheet opens showing that intent's details

**Test 6: Simulation updates (AGNT-06)**
Post a new intent and watch the simulation status.
Expected: "Simulation: Running..." with spinner, then "Simulation: Passed" (green check) or "Simulation failed" (red banner)

**Test 7: Approve flow (AGNT-07, AGNT-08)**
Tap "Approve" on a pending intent.
Expected:
  - Sheet transitions to progress view: "Signing..." with spinner
  - Then "Submitting..." with spinner
  - Then "Confirmed" with green check + truncated signature
  - Auto-dismiss after 3 seconds
  - If more pending: next intent slides in

**Test 8: Reject flow (AGNT-07)**
Tap "Reject" on a pending intent.
Expected:
  - Sheet closes immediately (or advances to next intent)
  - Badge decrements
  - Intent removed from queue

**Test 9: Agent status polling (AGNT-09)**
After approving an intent, poll its status:
```bash
curl http://localhost:9876/intent/INTENT_ID/status \
  -H "Authorization: Bearer YOUR_API_KEY"
```
Expected: Status progresses through "approved" -> "signing" -> "submitted" -> "confirmed" (with signature)

After rejecting:
```bash
curl http://localhost:9876/intent/INTENT_ID/status \
  -H "Authorization: Bearer YOUR_API_KEY"
```
Expected: `{ "status": "rejected" }`

**Test 10: sign_message intent (AGNT-03, AGNT-04)**
```bash
curl -X POST http://localhost:9876/intent \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"type": "sign_message", "message": "48656c6c6f20576f726c64"}'
```
Expected: Sheet shows with "Sign Message" heading, info banner "Message signature only. No transaction, no fees.", "Sign" button (not "Approve"), message as "Hello World" (UTF-8 decoded)

**Test 11: After idle lock (AGNT-05)**
Lock the app (idle or manual lock), then POST an intent.
Expected: Badge count increments but no auto-popup. After unlock, badge visible, user navigates to Agent API screen to see queue.
