## Spacing Scale

Declared values (multiples of 4). Sourced from existing widget measurements across welcome_step.dart, settings_screen.dart, app_shell.dart.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline spacing |
| sm | 8px | Between related items (e.g. label → field, password field → hint) |
| md | 16px | Standard element separation within a section |
| lg | 24px | Section padding (outer ListView padding matches settings_screen.dart) |
| xl | 32px | Outer screen padding for full-screen layouts (lock screen, onboarding step) |
| 2xl | 48px | Major visual gaps (logo → heading on lock screen and welcome step) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Password submit button height: minimum 44px (touch/click target; matches `ElevatedButton` with `vertical: 12` padding on a 20px font = 44px total)
- Strength meter bar height: 4px (xs) — intentionally slim decorative element

## Typography

Sourced from existing codebase: welcome_step.dart (28/16), settings_screen.dart (24/14), brand_theme.dart text roles. Four-size scale: 28 (display), 24 (heading), 16 (body), 14 (label + hint). Two-weight scale: 400 regular and 700 bold.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 700 (bold) | 1.2 | Lock screen heading ("Deadbolt"), onboarding step primary headline |
| Heading | 24px | 700 (bold) | 1.2 | Settings section heading ("Security"), screen title ("Settings") |
| Body | 16px | 400 (regular) | 1.5 | Lock screen tagline, password field labels, dialog body copy |
| Label / Hint | 14px | 700 (bold) for labels; 400 (regular) for hint text | 1.4 | Labels: settings category headers, form labels. Hint: strength meter state ("Weak"/"Fair"/"Strong"), inline error text, idle timeout description |

Weights in use: 400 (regular) and 700 (bold) only.

## Color

Sourced directly from `lib/theme/brand_theme.dart` — BrandColors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (Onyx Black) | `BrandColors.background` — lock screen background, full-screen scaffolds |
| Secondary (30%) | `#141414` (Graphite) + `#1E1E1E` (Charcoal) | `BrandColors.surface` / `BrandColors.card` — input fields, dialog surfaces, onboarding card backgrounds |
| Accent (10%) | `#F87040` (Solar Flare) | `BrandColors.primary` — reserved for: Unlock button, strength meter "Strong" indicator, focused input border, password show/hide icon active state |
| Destructive | `#E74C3C` (Ember Red) | `BrandColors.error` — wrong password shake state border tint, error snackbar, "Reset App" action in settings |

Accent `#F87040` reserved for:
1. Primary Unlock button background (ElevatedButton on lock screen)
2. Password strength meter fill when state is "Strong"
3. Focused state border on password TextField (already defined in brand_theme.dart InputDecorationTheme)
4. Active state of password visibility toggle icon

Accent is NOT used for: section headers, dialog titles, hint text, secondary buttons, idle timeout slider track.

Additional semantic colors (from BrandColors, used as needed):
- `#2ECC71` (Crypto Green) — strength meter "Fair" and "Strong" positive states (secondary indicator only)
- `#E2A93B` (Tungsten/Warning) — strength meter "Weak" state, delay countdown text

## Screen Layouts

### Lock Screen (`/lock`)

```
┌─────────────────────────────────────┐
│         #000000 background          │
│                                     │
│              [Logo]                 │  48px top padding from center anchor
│           80px height               │
│                                     │
│          "Deadbolt"                 │  Display / 28px / bold
│     "Your self-custodial            │  Body / 16px / #B0B0B0
│       Solana wallet"                │
│                                     │  48px gap
│  ┌─────────────────────────────┐    │
│  │  Password          [eye]    │    │  TextField, card fill, obscure=true
│  └─────────────────────────────┘    │
│                                     │  12px gap
│  [delay message if attempts > 0]    │  Label-Hint / 14px / #E2A93B / "Wait 5s..."
│                                     │  8px gap
│  ┌─────────────────────────────┐    │
│  │         Unlock              │    │  ElevatedButton, full width
│  └─────────────────────────────┘    │
│                                     │  Outer padding: 32px sides
└─────────────────────────────────────┘
```

- Logo: `assets/deadbolt_logomark.png` at 80px, `color: Colors.white, colorBlendMode: BlendMode.srcIn`
- Tagline: "Your self-custodial Solana wallet" — matches welcome_step.dart existing copy
- Password field: full-width, `obscureText: true`, visibility toggle suffix icon. Toggle icon: `Semantics(label: 'Show password')` when obscured, `Semantics(label: 'Hide password')` when visible.
- On wrong password: field border briefly shows `BrandColors.error`, widget applies `Tween<Offset>` shake (±8px horizontal, 200ms, 3 cycles)
- Escalating delay display: shown between field and button when `failedAttempts > 0`, format "Please wait {N}s before trying again"
- Loading state: Unlock button shows `CircularProgressIndicator(color: Colors.black, strokeWidth: 2)` in place of label text while `verifyAppPassword` async call is in-flight

## Interaction Contracts

### Escalating Delay on Wrong Password

| Attempt | Delay Before Next Try |
|---------|----------------------|
| 1 (first wrong) | 0s |
| 2 | 1s |
| 3 | 2s |
| 4 | 5s |
| 5 | 10s |
| 6+ | 30s |

- Delay applies to the lock screen only, not the auth challenge dialog
- Delay countdown shown as text: "Please wait {N}s before trying again"
- Unlock button disabled during countdown
- Attempt count resets to 0 on successful unlock

### Password Strength Meter States

| State | Trigger | Bar Fill | Bar Color | Label |
|-------|---------|----------|-----------|-------|
| Weak | length < 8 OR score < 2 | 33% | `#E2A93B` | "Weak" |
| Fair | length >= 8 AND score == 2 | 66% | `#2ECC71` | "Fair" |
| Strong | length >= 12 AND score >= 3 | 100% | `#F87040` | "Strong" |

Score: count of [hasUppercase, hasLowercase, hasDigit, hasSpecial] that are true.

### Lock Screen Transitions

- App launch → `/lock` (GoRouter redirect, instant, no animation)
- Idle timeout fires → lock screen overlaid via GoRouter redirect (`pushReplacement` to `/lock`) with 150ms fade transition
- Correct password entered → fade to `/dashboard`, 200ms, `Curves.easeOut`
- Lock screen to lock screen (already on `/lock`, idle fires again) — no-op

### Auth Challenge Dialog

- Triggered by: tap "Reveal Seed Phrase", tap "Create API Key", tap "Delete API Key"
- Phase 4 guardrail bypass also uses this dialog (hook already defined)
- Dismiss (Go Back): action is abandoned, no state change
- Confirm (Verify correct password): action proceeds, dialog dismisses
