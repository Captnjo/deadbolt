       Change `decrypt_with_key()` signature from:
       ```rust
       pub fn decrypt_with_key(...) -> Result<Vec<u8>, DeadboltError>
       ```
       To:
       ```rust
       pub fn decrypt_with_key(...) -> Result<Zeroizing<Vec<u8>>, DeadboltError>
       ```
       Wrap the return: `Ok(Zeroizing::new(plaintext))` instead of `Ok(plaintext)`.

       Update existing tests in vault.rs that call decrypt/decrypt_with_key — the return type is now `Zeroizing<Vec<u8>>` which `Deref`s to `Vec<u8>`, so most `.as_slice()` calls will still work. If any test does `decrypted == seed`, change to `decrypted.as_slice() == &seed` or `*decrypted == seed.to_vec()`.

    2. **session.rs** — Update callers of vault::decrypt_with_key:

       In `unlock_with_password()`: Same — `plaintext` from `vault::decrypt()` is now `Zeroizing<Vec<u8>>`. Auto-zeroes. No code changes needed beyond accepting the new type (Deref handles it).

       - No changes needed to mnemonic.rs. Document this finding in the SUMMARY.

    4. **wallet_manager.rs** — Update callers of vault::decrypt_with_key:
       In `get_mnemonic()`: The `plaintext` from `vault::decrypt_with_key()` is now `Zeroizing<Vec<u8>>`. The `String::from_utf8(plaintext)` call needs adjustment because `from_utf8` takes `Vec<u8>`, not `Zeroizing<Vec<u8>>`. Change to:
       ```rust
       let plaintext = vault::decrypt_with_key(&vault_data, &vault_key)?;
       let phrase = String::from_utf8(plaintext.to_vec())
           .map_err(|e| DeadboltError::VaultError(format!("Invalid mnemonic data: {e}")))?;
       ```
       Note: `plaintext.to_vec()` creates a copy, but `plaintext` (Zeroizing) is dropped right after, zeroing the original. The `phrase` String is returned to the caller. This is acceptable — the mnemonic phrase is the intended output.

       Alternatively, use `String::from_utf8((*plaintext).clone())` or `Zeroizing::into_inner()` if available — but `to_vec()` is clearest.
