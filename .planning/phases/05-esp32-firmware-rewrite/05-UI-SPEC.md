## Spacing Scale

Declared values (multiples of 4). Source: observed across existing screens (`connect_hardware_screen.dart`, `display_mnemonic_step.dart`, `agent_api_screen.dart`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, word-number label gap in MnemonicGrid (`SizedBox(width: 4)`) |
| sm | 8px | Card spacing, chip wrap spacing (`spacing: 8`, `runSpacing: 8`) |
| md | 16px | Default container padding, section dividers |
| lg | 24px | Screen content padding (`padding: all(24)`), horizontal button padding |
| xl | 32px | Full-screen content padding (`padding: all(32)`) |
| 2xl | 48px | Major section breaks (loading states, USB icon empty states) |
| 3xl | 64px | Page-level spacing (not currently used; reserved) |

**Locked legacy exceptions** (pre-existing codebase values — do not change without a dedicated refactor task):

| Location | Value | Source file | Notes |
|----------|-------|-------------|-------|
| Wallet drawer tile padding | `symmetric(horizontal: 20, vertical: 10)` | `lib/shared/widgets/wallet_drawer.dart:128` | 20px horizontal and 10px vertical are non-multiples-of-4; inherited from existing drawer implementation |

## Typography

Source: `brand_theme.dart` TextTheme + observed sizes across existing screens.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (normal) | 1.5 |
| Label | 12px | 400 (normal) | 1.4 |
| Heading | 24px | 700 (bold) | 1.2 |
| Subheading | 16px | 700 (bold) | 1.3 |

Two weights only: 400 (normal) and 700 (bold). Subheading is distinguished from body by size (16px vs 14px), not by a separate weight. Do not introduce w600 or any third weight in Phase 5 code.

**Monospace exception:** All pubkey/address strings and mnemonic words use `fontFamily: 'monospace'` at 12–13px weight 400. Apply consistently on:
- Pubkey display in device dashboard
- Mnemonic word chips in `MnemonicGrid`
- Port path in device detection list

## Screen Inventory

Phase 5 introduces three new Flutter screens and modifies one existing widget. All inherit `buildBrandTheme()`.

### 1. Hardware Wallet Sidebar Entry (modify `wallet_drawer.dart`)

**States (3):**

| State | Visual |
|-------|--------|
| Not paired | USB outlined icon (`Icons.usb_outlined`, 18px, `BrandColors.textSecondary`), label "Hardware Wallet", trailing Badge with text "Setup" in `BrandColors.primary` background |
| Paired + disconnected | USB icon (`Icons.usb`, 18px, `BrandColors.textDisabled`), device name, no badge |
| Connected | USB icon (`Icons.usb`, 18px, `BrandColors.primary`), device name, green status dot (8px circle, `BrandColors.success`) |

**Tap target:** Full drawer tile height minimum 48px (accessibility touch target).
**Entry point:** Tapping navigates to `/hardware` in all states. When unpaired, dashboard shows setup guide flow.

### 2. Hardware Wallet Screen (`lib/features/hardware/hardware_wallet_screen.dart`)

**Layout:** `Scaffold` with `AppBar` (title "Hardware Wallet"), `body: Center(child: ConstrainedBox(maxWidth: 600, ...))` — mirrors `connect_hardware_screen.dart` pattern.

**Sub-states:**
- **Not paired:** Setup guide flow — vertically stacked steps card, "Connect Device" ElevatedButton at bottom
- **Paired + disconnected:** Device info card (name, truncated pubkey in monospace, status "Disconnected" in `BrandColors.textSecondary`), "Reconnect" OutlinedButton, "Factory Reset" destructive OutlinedButton with `BrandColors.error` foreground
- **Connected:** Device info card (name, truncated pubkey, green status dot + "Connected"), action buttons: "Generate New Keypair" (ElevatedButton, `BrandColors.primary`), "Disconnect Device" (OutlinedButton), "Factory Reset" (OutlinedButton, `BrandColors.error` foreground)
- **Pubkey mismatch:** Persistent warning banner above device info card — `Container` with `BrandColors.error.withAlpha(15)` fill, `BrandColors.error.withAlpha(60)` border, `Icons.warning_amber_rounded` 18px + message text. Action buttons: "Re-register" (ElevatedButton) and "Disconnect Device" (OutlinedButton). Signing blocked (no generate action shown).

### 3. Mnemonic Display Screen (`lib/features/hardware/mnemonic_display_screen.dart`)

**Screenshot prevention:** `NoScreenshot.instance.screenshotOff()` on `initState()`, `screenshotOn()` on `dispose()`. Banner text: "Screenshot blocked — write down your words."

**Layout:** `SingleChildScrollView` with `padding: all(32)` — mirrors `display_mnemonic_step.dart`.

**Structure:**
1. Heading "Your Recovery Phrase" — 24px bold
2. Subtext — 14px `BrandColors.textSecondary`
3. Screenshot-blocked notice — `SecurityTip` widget at `SecurityTipLevel.critical`
4. `MnemonicGrid(words: words)` — reuse existing widget, warning border (`BrandColors.warning.withAlpha(80)`)
5. Two `SecurityTip` widgets (critical + warning levels) — reuse existing widget
6. CTA: "I've Written It Down" — `ElevatedButton` full width

**No copy-to-clipboard button** — this is the hardware wallet flow where the mnemonic must be written down, not copied. (The onboarding flow has copy; this flow explicitly omits it per security design.)

### 4. Mnemonic Quiz Screen (`lib/features/hardware/mnemonic_quiz_screen.dart`)

**Layout:** `SingleChildScrollView` with `padding: all(32)` — mirrors `verify_backup_step.dart`.

**Structure:**
1. Heading "Verify Your Backup" — 24px bold
2. Subtext — 14px `BrandColors.textSecondary`
3. 3–4 `_QuizQuestion` widgets (reuse `VerifyBackupStep._QuizQuestion` pattern or extract to shared widget)
4. Error container when wrong answer: `BrandColors.error.withAlpha(15)` fill, `Icons.error_outline` — reuse `verify_backup_step.dart` pattern exactly
5. No explicit CTA button — quiz auto-advances when all questions correct (same as onboarding pattern)

## Interaction Contract

### Mnemonic display — one-shot enforcement

Words are shown exactly once. After the user taps "I've Written It Down" and passes the quiz:
1. Quiz auto-advances to device dashboard on all-correct
2. Flutter provider holding words is immediately invalidated (`ref.invalidate()`)
3. Screen cannot be navigated back to (use `context.go()` not `context.pop()`)

### Factory reset — two-factor gate

1. User taps "Factory Reset" button
2. `showAuthChallengeDialog(context)` fires — password re-entry required
3. On auth success: confirmation dialog appears with copy above
4. On dialog confirm: app sends reset command to firmware
5. Firmware displays progress; user must hold BOOT button 5 seconds
6. Success: `SnackBar` "Hardware wallet reset. Device unpaired." + navigate to empty-state dashboard
7. Failure/timeout: `SnackBar` "Reset failed — try again" in `BrandColors.error` background

### Disconnect notification

`ScaffoldMessenger.showSnackBar` with:
- `content: Text('Hardware wallet disconnected')`
- `duration: Duration(seconds: 3)`
- `backgroundColor: BrandColors.error`
- Source: established `BrandColors.error` snackbar pattern from RESEARCH.md

### Generation loading state

Generation (PBKDF2 + SLIP-0010 on ESP32) takes up to 30 seconds. During generation:
- Button replaced with `CircularProgressIndicator(strokeWidth: 2)` inside 20×20 `SizedBox` (existing pattern from `connect_hardware_screen.dart`)
- Status label updates to "Generating keypair..."
- Navigation locked (no back gesture during generation)

### Pubkey mismatch banner

- Persistent `Container` banner pinned above device info card — not dismissable
- Signing/generate actions hidden; only "Re-register Device" and "Disconnect Device" shown
- Banner persists until user takes one of the two resolution actions
- Re-register replaces stored pubkey with newly detected pubkey (after auth challenge)
